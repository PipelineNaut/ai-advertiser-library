🔌 Before The Week Starts — The Meta Connection (Read This)

Before you run a single prompt in this lesson, you need Claude wired into Meta. Half the questions in the community right now come from students who got the install wrong because they confused two completely different Meta artifacts. Let's clear that up first.

Meta dropped TWO things — you only need one

On April 29, 2026, Meta released both at the same time. They have similar names. They are NOT the same product.

🔵 The Meta Ads MCP

What 99% of you need
A hosted HTTP endpoint at https://mcp.facebook.com/ads. No install. You point your AI tool at this URL, log into Facebook once, you're connected forever. This is what Claude Desktop and Claude Code (the AI you talk to) use.
✓ This is what this lesson uses

⚙️ The Meta Ads CLI

For engineers only
A standalone command-line binary. You install it locally with npm i -g @meta/ads-cli and call it from bash scripts, cron jobs, or Python pipelines. Useful if you're building backend automation outside of an AI chat. Most students never need this.
✗ Skip unless you're scripting

💡 The translation: When somebody says "Meta CLI" they usually mean the MCP (because they're calling the install command "the CLI"). Officially Meta calls the URL approach the MCP server and the binary approach the CLI. For everything in this lesson, the MCP is the right answer.

🖥️ Path A — If you use Claude Desktop

Claude Desktop is the standalone Claude app on Mac/Windows. Click → click → chat. No terminal, no VS Code, no command lines anywhere.

CLAUDE DESKTOP

5-click install

Zero terminal. Zero config file editing.
  1. Open Claude Desktop
  2. Settings (cog icon, bottom left) → Connectors
  3. Click the + Add custom connector button
  4. Name: META MCP  ·  URL: https://mcp.facebook.com/ads
  5. Click Add → approve Facebook OAuth in browser → done
Result: 29 Meta Ads tools appear under the "Tools" section of Claude Desktop. Type "list my ad accounts" in any chat and Claude does the rest.

💻 Path B — If you use Claude Code inside VS Code

Claude Code is the agent that lives in your terminal. Most AI Advertiser students run it inside VS Code's integrated terminal or as a VS Code panel. Two ways to install — pick whichever feels less scary.

VS CODE — WAY 1 (EASIEST)

Let Claude install it for you

No terminal commands. You just paste a prompt and Claude handles everything.
  1. Open VS Code in any project folder
  2. Start Claude Code — either via the panel extension OR by opening the terminal (Cmd+`) and typing claude
  3. Paste the prompt below into the chat
  4. When Claude asks, log into Facebook and approve the connection
Install the official Meta Ads MCP for me. It's an HTTP transport server at https://mcp.facebook.com/ads — please run the "claude mcp add" command with the http transport. After it's installed, test the connection by listing my Facebook ad accounts and walk me through the OAuth approval flow step by step.
Result: Claude runs the install command, opens Facebook OAuth in your browser, waits for you to authorize, then confirms by listing your ad accounts. You never touched the terminal directly.
VS CODE — WAY 2 (MANUAL)

Run the command yourself

Use this if Way 1 throws errors or your Claude Code version is stubborn about installs.
  1. Open VS Code terminal: Cmd+` (Mac) or Ctrl+` (Windows)
  2. Paste this command and hit enter:
$ claude mcp add --transport http meta-ads "https://mcp.facebook.com/ads"
  1. Start Claude Code: type claude and hit enter
  2. Paste any Meta-related prompt (e.g. "list my ad accounts") — the OAuth flow triggers on first use
Result: Same end state as Way 1. The only difference is YOU typed the install command instead of letting Claude do it.

⚠️ The setup is per-app, not per-account. If you install it in Claude Desktop and ALSO use VS Code's Claude Code, you'll authorize Facebook OAuth twice — once for each app. That's normal. After those two one-time approvals, both apps stay connected independently.

❓ Common confusions cleared up

Do I need to install BOTH Claude Desktop and VS Code?

No. Pick one based on your workflow. The MCP works the same in either. Most students who go past $5k/mo end up using both — Desktop for quick chats, VS Code for the daily ops grind.

I saw "@meta/ads-cli" on GitHub — is that what I install?

No. That's the standalone CLI binary for scripts and cron jobs. For everything in this course you want the MCP (the URL approach). Installing the CLI doesn't connect Claude to Meta — those are two different products that happened to ship the same day.

Will my Facebook session expire and break this every few hours?

No. That was the OLD token-based API (which is why we killed the old lesson 5.3). The MCP uses Meta-managed OAuth tokens that are refreshed server-side. You don't see them, you don't refresh them, they don't expire on you.

Can I use this with multiple Facebook accounts (one per client)?

Yes — but it's per-Facebook-profile. When you OAuth, you grant access to whatever ad accounts that profile can see (your own + any client BMs that gave you access). To use a totally different personal profile, sign out and re-authorize.

After install, where do I "see" the MCP is connected?

Claude Desktop: Settings → Connectors → you'll see "META MCP" with a green dot.  ·  Claude Code: run claude mcp list in your terminal — meta-ads should appear in the output.

Do I need a Meta Dev App, Marketing API approval, or any of that old garbage?

No. None of it. That's what made the old setup miserable. The official MCP bypasses the entire Dev App world — you just OAuth as yourself like you're logging into any other Facebook-connected app.

I installed it, but Claude isn't using it when I ask Meta questions. Why?

Two common causes: (1) you installed it in the WRONG Claude — e.g. you set it up in Claude Desktop but you're chatting in VS Code. The MCP is per-app. (2) Claude doesn't know to use it yet — just say in chat: "List your available MCP tools — I want to confirm meta-ads is loaded." If it's listed, your next Meta prompt will trigger it. If it's not, the install didn't complete.

⚠️ Don't combine install paths without a reason. Some students panic, install Path A, then ALSO run Path B's manual command "just to be safe." This doesn't break anything but it creates two separately-authorized connections you have to manage. Pick one path, finish it, test it works, move on.

Once you confirm Claude can list your ad accounts, you're ready. The rest of this lesson assumes the MCP is live. Now let's get into the actual 7-day routine.

📅 Day 1 — Monday

1
Monday

Audit & Diagnose

⏱ 30 min 🎯 Find what's working
Walk into Monday knowing exactly which campaigns deserve more money, which need to die, and which are quietly bleeding cash. No spreadsheets, no Ads Manager rabbit holes. One prompt → full picture.

The morning prompt

Pull last 30 days of performance for ad account [ACCOUNT_ID]. Build me a table sorted by ROAS descending with these columns: campaign name, status, spend, CPA, ROAS, CTR, frequency, hook rate. Then below the table give me 3 lists:

1. SCALE — campaigns with ROAS above target and frequency under 2.5
2. KILL — campaigns with CPA above target after $100+ spend OR ROAS below 1.0
3. INVESTIGATE — campaigns with high spend but unclear performance (frequency above 3, CTR dropping, etc.)

Be opinionated. Don't just dump data — tell me what to do.

The deep work block (15 min)

After Claude gives you the kill list, drill into the WHY before executing. Bad audience? Creative fatigue? Landing page mismatch? Send this:

For each campaign in the KILL list, dig deeper. Look at: creative-level CTR breakdown, audience overlap with my scaling campaigns, day-by-day performance trend, and frequency curve. Tell me whether the campaign died because of creative fatigue, audience saturation, landing page mismatch, or just a bad concept. Be specific.
Expected outcome

A clear list of campaigns to kill, scale, or investigate — with the reasoning behind each call. You walk into Tuesday knowing what creative concepts are dying and need replacement.

Common mistake

Don't just take Claude's KILL recommendations and pause everything immediately. Read the WHY first. If it's creative fatigue, the audience is still good — keep that ad set, just swap creative. Killing prematurely throws away learning.

📅 Day 2 — Tuesday

2
Tuesday

Creative Generation

⏱ 45 min 🎯 Build the next batch
Yesterday told you what's dying. Today you replace it. Pull the patterns from your winners, generate 15-25 new hook variations, brief 5 fresh angles. Volume wins. The agencies stuck on 2 creatives a week are why they lose.

Step 1 — Mine your winners (10 min)

Look at the top 5 ROAS ads in account [ACCOUNT_ID] from the last 60 days. Pull their full ad copy (primary text, headline, description) and analyze what they have in common: hook structure, emotional driver (curiosity / fear / aspiration / proof), CTA style, length, format (single image / carousel / video). Give me a 'winning ad DNA' summary I can use to brief new creative.

Step 2 — Generate the new batch (20 min)

Based on the winning ad DNA you just identified, write 15 new ad variations for [CLIENT_NAME / OFFER_NAME]. Split them like this:

- 5 hooks using the SAME proven structure but new angles
- 5 hooks testing a NEW emotional driver (whichever wasn't represented in the winners)
- 5 hooks that are deliberately weird/contrarian to break category norms

Each variation gives me: primary text, headline (under 40 chars), and description. Match the client's voice — casual, no AI patterns, no em dashes, no triple structures. Write like a real human typing in IG comments.

Step 3 — Image / video briefs (15 min)

For each of the 15 ad copy variations, write a 1-line creative brief that I can hand to a designer or feed into image gen. Specify: format (static / carousel / video), main visual element, mood, text overlay (if any), and one reference style (e.g. 'Hormozi school dark + bold text', 'soft pastel UGC', 'iPhone screenshot style', 'meme/contrarian'). Keep each brief under 30 words.
Expected outcome

15 fresh ad copy variations + 15 creative briefs ready to hand off. You're not testing 2 ads per week like an agency on training wheels — you're testing 15.

Common mistake

Don't skip Step 1. Generating creative without mining your winners is the #1 reason AI-written ads underperform. The DNA pass turns Claude from a random idea generator into a pattern-matched copywriter.

📅 Day 3 — Wednesday

3
Wednesday

Build & Launch

⏱ 40 min 🎯 Ship the new batch
Take Tuesday's 15 variations and push them into Meta. The MCP can build campaigns, ad sets, and ads in PAUSED state in seconds. Your only job is to QA before flipping the switch.

Build everything paused (25 min)

Build a new campaign in account [ACCOUNT_ID] using this exact structure:

CAMPAIGN
- Name: [#XXX] [PAGE-A] [LEADS] [TEST-COLD] [1-3-1 ABO] [BATCH-MAY-15] — 05/15
- Objective: leads (or sales for ecom)
- Buying type: AUCTION
- Status: PAUSED

AD SETS (create 3, duplicates of each other)
- Daily budget: $50 each
- Audience: [SPECIFY: lookalike / interest / broad]
- Placement: Advantage+ placements
- Optimization: leads (or purchase)

ADS (create 5 in each ad set, same 5 across all 3 ad sets)
- Use the 5 BEST variations from yesterday's batch
- Use Page ID [PAGE_ID] and Pixel ID [PIXEL_ID]
- Apply UTM parameters per Brez convention: utm_source=meta&utm_medium={{placement}}&utm_campaign={{campaign.name}}&utm_content={{adset.name}}&utm_term={{ad.name}}

Show me the structure preview before creating. Build everything in PAUSED state. I'll activate manually after QA.

QA check (10 min)

Before I activate, run a QA pass on the campaign you just built. Check: budgets sum correctly, all ads have the right Page + Pixel, UTM parameters are applied at the ad level, optimization event matches the objective, audience size is between 500K and 5M (warm flag if outside), no typos in copy, headline length under 40 chars. Flag anything wrong. Don't activate yet.

Activate (5 min)

QA passed. Pull the trigger:

QA looks good. Activate the campaign and all ad sets you just built. Then confirm back: campaign ID, ad set IDs, and that all are ACTIVE.
Expected outcome

New campaign live with 3 ad sets × 5 ads = 15 active ad creatives testing. Total daily spend: $150. By Thursday morning you'll have ~24 hours of data to act on.

⚠️ Never let Claude activate without QA. The MCP can build campaigns in seconds. It can also activate $5,000/day campaigns in seconds. Always run the QA prompt and read the preview before saying go.

📅 Day 4 — Thursday

4
Thursday

Optimization

⏱ 25 min 🎯 Cut losers, scale winners
24-48 hours of data on Wednesday's launch. Time to be ruthless. Kill what isn't working before it burns more spend. Double down on what is.

The killer prompt (10 min)

Look at the campaign launched on Wednesday in account [ACCOUNT_ID]. For each ad with at least $30 in spend, check: CPA vs target (target is $XX), ROAS vs target (target is X.X), hook rate, CTR. List me every underperformer and recommend whether to pause it. Show me the list first and wait for my confirmation before pausing anything.

The scaler prompt (10 min)

Now show me the winners — any ad with CPA below target and ROAS above target after at least $30 spend. For each winner: duplicate the ad set with 2x the budget into a new campaign with the same structure but PAUSED. This is my scale stack — I'll activate manually tomorrow once today's data fully confirms the win. Never auto-scale.

Anti-fatigue check (5 min)

Check all my currently active scaling campaigns in account [ACCOUNT_ID]. For each one, look at frequency over the last 7 days. Flag any ad set where frequency is above 2.8 OR where CTR has dropped more than 25% week-over-week. These are showing fatigue — recommend either refreshing creative or pausing.
Expected outcome

Losers paused. Winners queued at 2x budget for tomorrow's activation. Fatigue signals caught before they tank ROAS. The account is leaner and more profitable than yesterday.

Common mistake

Don't scale on day 1. New campaigns need 48-72 hours to leave the learning phase. Activating the 2x duplicates the same day you spot a winner often kills the original campaign's pacing. Patience compounds.

📅 Day 5 — Friday

5
Friday

Client Reporting

⏱ 30 min 🎯 Make the client feel like a genius
Client retention is 80% about communication, 20% about results. Friday is when you turn the week into a narrative they can forward to their business partner. This single prompt replaces 2 hours of dashboard wrangling.

The weekly report prompt (20 min)

Generate a client-ready weekly report for [CLIENT_NAME] (account [ACCOUNT_ID]) covering the last 7 days. Format it as plain markdown I can paste into Loom notes or an email. Structure:

1. HEADLINE METRICS — spend, leads/sales, CPA, ROAS, vs last week (with % change)
2. THE STORY — 2-3 sentences explaining what happened this week in plain English (not jargon)
3. WHAT WORKED — 2-3 specific wins (top ad concept, audience that hit, scaling move)
4. WHAT DIDN'T — 1-2 misses and what we learned
5. NEXT WEEK'S MOVE — 2-3 concrete actions we're taking based on this week's data

Write it like a smart operator updating a partner — direct, no fluff, no apologizing for losses. Numbers first, story second.

The Loom prep (10 min)

Drop the report into Loom notes, screen-record yourself walking through it for 3-5 minutes. Send the Loom link. Clients pay for the feeling that you understand their business, not the dashboard itself.

Expected outcome

One Loom + report sent every Friday before EOD. Clients stop ghosting you, retention goes from 4 months to 12+, and they refer you to their network unprompted.

💡 Make this non-negotiable. The agencies that lose clients are the ones that go silent for 2 weeks and only show up to ask for an invoice approval. A weekly Friday Loom is the cheapest insurance policy in your business.

📅 Day 6 — Saturday

6
Saturday

Research & Intel

⏱ 45 min 🎯 Build the edge for next week
The accounts that compound year over year aren't the ones running the best ads — they're the ones with the best intel pipeline. Saturday is where you spy on competitors, mine VOC, and seed Monday's hypotheses.

Competitor sweep (20 min)

I want a competitor sweep for [CLIENT_NIICHE]. Pull the top 5 advertisers in this niche from the Meta Ad Library (search for [SEED_KEYWORDS]). For each one: how many ads they have running, the most common hook angle, the format mix (video vs static vs carousel), the offer they're leading with, and any pricing they're showing publicly. End with: 'here are 3 angles they're using that we're NOT testing — recommend one we should adapt for next week.'

Voice-of-customer mining (15 min)

Scrape Reddit, YouTube comments, and Amazon reviews for [CLIENT_NIICHE / PRODUCT]. Pull 20-30 quotes that include emotional language about the problem we solve. Group them into 3 buckets: PAIN (what hurts), DESIRE (what they want), OBJECTION (why they haven't bought yet). I want to use this as raw material for next week's hooks.

Next-week hypothesis (10 min)

Take everything from today (competitor sweep + VOC mining) and synthesize: what's the ONE hypothesis we should test next week that nobody else in this market is testing? Write it as a tweet-length statement I can put at the top of Monday's plan.
Expected outcome

Fresh competitor intel + 20-30 VOC quotes + one sharp hypothesis. Monday no longer starts cold — you walk in with a thesis to validate.

📅 Day 7 — Sunday

7
Sunday

Plan & Reset

⏱ 20 min 🎯 Compound the system
Most operators don't take Sunday seriously and that's why they stay stuck doing the same week over and over. 20 minutes here = the difference between an agency that plateaus and one that compounds.

The retrospective (10 min)

Run a retrospective on the last 7 days for account [ACCOUNT_ID]. Tell me:

1. What was the single biggest win and what made it work?
2. What was the single biggest miss and what would I do differently?
3. What SOP or workflow tweak would have made this week 20% faster?
4. What's one assumption about this account I held on Monday that I no longer believe on Sunday?

Be brutal. No coddling. I want the operator-level read, not a participation trophy.

Update your SOPs (5 min)

If the retrospective surfaced a workflow improvement, capture it now while it's fresh:

Based on this week's retrospective, write a 3-5 line addendum to my operator SOP that I can add under the relevant day. Make it specific, actionable, and dated.

Step away (5 min)

Close the laptop. Go outside. The system runs while you're not at the keyboard. Operators who can't unplug burn out at $15k/mo and never make it to $50k. Build the business that lets you actually live.

Expected outcome

One sharper week ahead. Your SOPs get a little better. Your hypotheses get a little better. Over 52 Sundays this is the difference between a $10k/mo operator and a $100k/mo operator running 12 accounts in 4 hours a day.

🧠 The Meta-Lesson

Look at the structure of this week. Monday looks at the past. Tuesday-Wednesday builds the future. Thursday adjusts. Friday communicates. Saturday spies. Sunday compounds.

Every day has one job. You're not multitasking — you're stacking single-purpose 20-45 minute blocks across a week. That's why a solo operator with this workflow runs circles around a 5-person team without one.

The MCP didn't make you 10x faster. The MCP plus a structured weekly system made you 10x faster. Most students get the tool, skip the system, and wonder why nothing changes.

💡 One client first. Run this routine for ONE account for 4 weeks straight before adding a second client. The compounding effect doesn't show up in week 1. It shows up in week 4 when you realize you're doing it on autopilot and have hours left in the day.
