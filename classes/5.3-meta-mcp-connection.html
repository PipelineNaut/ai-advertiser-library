Lesson 5.3 — Module 5: AI Integration

Connect Claude Code to Meta (Official MCP)

~5 min No tokens No code Updated April 2026
Big update. Meta dropped the official MCP server in late April 2026. The old way (manual access tokens, refresh every 2 hours, Dev App approval, custom API code) is dead. Now you paste one prompt into Claude Code inside VS Code, log in with Facebook once, and you're connected forever. No terminal commands. No setup wizards. This lesson takes ~5 minutes.

🚨 What Changed (Read This First)

If you've been in the program for a while, you remember the painful old way: generate a long-lived access token, paste it into a config file, pray it doesn't expire, regenerate it every 2 hours when it did. That whole flow is gone.

On April 29, 2026, Meta released the official Meta Ads MCP server. It's first-party Meta software (not built by a community), and it does the boring auth stuff for you.

❌ Old way (dead)

  • Create a Facebook Dev App
  • Wait for Marketing API approval
  • Generate long-lived access token
  • Paste into a config file
  • Token expires every ~2 hours
  • Re-generate constantly
  • Write custom API code for every action

✅ New way (this lesson)

  • Paste 1 prompt into Claude Code in VS Code
  • Log in with Facebook (OAuth) once
  • Done. Forever.
  • No terminal commands
  • No tokens to manage, no expiry
  • 29 prebuilt tools ready to use
  • Just talk to Claude in plain English

💡 Translation: The thing that used to take 20 minutes and break every other day now takes 5 minutes and never breaks. This is one of the biggest quality-of-life upgrades since we started this program.

⚡ The Whole Setup (Paste 1 Prompt in VS Code)

This entire install happens inside VS Code. You don't open Terminal. You don't run setup wizards. You paste one prompt into Claude Code and the assistant handles the install, the verification, and the Facebook authorization flow for you.

Open Claude Code inside VS Code

VS Code → open your project folder → click the Claude icon (left sidebar)

Open your agency project folder in VS Code. On the left sidebar, click the Claude Code icon to open the chat panel. If you don't have a project folder yet, just open any folder in ~/Documents — Claude needs to be running inside a folder.

Paste this prompt into the Claude Code chat

Copy the full prompt below and paste it into the Claude Code chat box. Hit enter.

install the meta ads mcp server for me.

run this command in the terminal:
claude mcp add --transport http meta-ads "https://mcp.facebook.com/ads"

then run `claude mcp list` and show me the result.

after that, call a meta ads tool — try listing my ad accounts. when you do, vs code is going to pop up a login prompt asking me to authorize with facebook. i'll click through it.

once i'm authenticated, fetch my ad accounts and confirm everything's connected.

Watch Claude run the install — then click "Allow" when prompts appear

Claude will run two commands for you in sequence. VS Code will show permission prompts before each one — click Allow on both:

✓ Added MCP server meta-ads (http)
  URL: https://mcp.facebook.com/ads
  Authentication required on first use.

Then Claude will try to call a Meta Ads tool. That triggers the OAuth flow.

Authorize with Facebook in the browser

VS Code pops open a Meta OAuth screen in your default browser. You'll see the permissions Meta is granting:

  • ads_management — create & edit campaigns
  • ads_read — read performance data
  • business_management — see ad accounts inside your BM
  • catalog_management — for e-comm catalog ads (optional)

Click "Continue as [Your Name]". Pick which businesses/ad accounts to grant access to (just yours if you're solo, all of them if you manage clients). Click "Save". The browser tab will say "Connection successful, you can close this tab."

Back in Claude Code — see your accounts listed

Switch back to VS Code. Within a few seconds Claude will respond with your ad accounts in a table:

💡 You're done. No token. No expiry. No config file. No terminal commands you had to type yourself. Claude can now read and act on every account on that list. The OAuth handshake is stored securely on Meta's side — your next Claude Code session will just work, no re-authorization needed.

💬 Your First Real Prompts (Copy & Paste)

Don't read about it. Use it. Paste these into Claude Code one at a time and watch what happens. 

Show me the last 7 days of performance for every active campaign in ad account [PASTE ACCOUNT ID]. Sort by ROAS descending. Flag any campaign with CPA above $50 or ROAS below 1.5.
Look at ad account [PASTE ACCOUNT ID]. Find the top 3 ad creatives from the last 30 days based on ROAS. Tell me what they have in common (format, hook style, CTA) and suggest 5 new creative angles based on what's working.
Pause every ad set in account [PASTE ACCOUNT ID] that has spent more than $100 in the last 3 days with a ROAS below 1.0. Show me the list first and wait for my confirmation before pausing.
Create a new campaign in ad account [PASTE ACCOUNT ID] for client [CLIENT NAME]. Objective: leads. Daily budget: $50. Audience: US, age 25-55, interested in home services. Use the existing Page [PAGE ID] and pixel [PIXEL ID]. Set it to status PAUSED so I can review before activating.

⚠️ Always tell Claude to "set it to PAUSED" or "show me the list first and wait for confirmation" when you're making changes that affect spend. The MCP can move real money. Treat it like a junior media buyer with API access — capable, fast, but you check the work before it goes live.

🔧 Troubleshooting

No Facebook login window popped up

The OAuth prompt only fires when Claude actually calls a Meta Ads tool. If Claude finished the install and then stopped, paste this follow-up into the chat:

please call a meta ads tool right now to trigger the facebook authentication prompt. list my ad accounts.

The auth window should pop up immediately after.

"MCP server meta-ads already exists" or status stays "Failed to connect"

You may have installed it before. Tell Claude to reset it — paste this into the chat:

remove and reinstall the meta ads mcp server. run:
claude mcp remove meta-ads -s local
claude mcp add --transport http meta-ads "https://mcp.facebook.com/ads"

then trigger authentication by listing my ad accounts.

"claude: command not found" (when Claude tries to run the install)

The Claude Code CLI isn't installed or isn't on your PATH. Go back to Lesson 5.2 and finish the install. If you already installed it, close VS Code completely and reopen it — sometimes the PATH doesn't refresh in the integrated terminal until VS Code is restarted.

"This Facebook account doesn't have access to any Business"

You logged in with a personal Facebook profile that isn't tied to a Business Manager. Either log out and log in with the right profile, or go to business.facebook.com and create a BM first (see Lesson 3.4).

"Insufficient permissions" when Claude tries to do something

Most likely: during OAuth you unchecked a permission you actually need, or the Facebook profile you authorized doesn't have admin/advertiser role on the ad account in question. Re-authorize by pasting this into Claude Code:

reconnect the meta ads mcp. run:
claude mcp remove meta-ads -s local
claude mcp add --transport http meta-ads "https://mcp.facebook.com/ads"

then list my ad accounts to trigger the facebook authorization screen again. this time i'll make sure all permission boxes are checked.

Claude doesn't seem to know the MCP exists

Ask Claude to verify the registration — paste this into the chat:

run `claude mcp list` and show me whether meta-ads is registered. also list the meta-ads tools you currently have available.

If meta-ads isn't in the list, paste the install prompt from Step 2 again. If it's there but tools aren't loaded, close and reopen the Claude Code panel in VS Code.

Want to do it the old terminal way?

If you'd rather skip the chat prompt and run things directly in the terminal — open the VS Code integrated terminal (View → Terminal or Ctrl + `) and run:

$ claude mcp add --transport http meta-ads "https://mcp.facebook.com/ads"

Then start the CLI with claude and type /mcp to authenticate. This is the manual path — the paste-prompt method above is what we recommend for everyone else.
